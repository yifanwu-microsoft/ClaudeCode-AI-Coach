Project internationalization (i18n) management tool. Detects translation sync status, synchronizes outdated translations, and adds new language support.

Arguments: $ARGUMENTS

## Argument Parsing

Determine execution mode based on `$ARGUMENTS`:

- **Empty** (no arguments) → Run "Status Check"
- **`sync`** → Sync all outdated translations across all languages
- **`sync <lang>`** (e.g., `sync en`) → Sync only the specified language
- **`add <lang>`** (e.g., `add ja`) → Add new language support

---

## File Mapping Table

Mapping between source files (Chinese, root directory) and translation files:

| Source File | Translation File |
|-------------|-----------------|
| `CLAUDE.md` | `<lang>/CLAUDE.md` |
| `PROGRESS.md` | `<lang>/PROGRESS.md` |
| `ai-engineering-leveling-guide.md` | `<lang>/ai-engineering-leveling-guide.md` |
| `.claude/commands/coach/assess.md` | `<lang>/commands/coach/assess.md` |

| `.claude/commands/coach/progress-report.md` | `<lang>/commands/coach/progress-report.md` |
| `.claude/commands/coach/review-prompt.md` | `<lang>/commands/coach/review-prompt.md` |

**README special handling**:
- `README.md` (English) is the README source file
- `README.zh.md` (Chinese), `README.<lang>.md` (other languages) are translations

## Language Discovery

Execute the following steps to discover supported languages:

1. Scan subdirectories in the root for those containing `CLAUDE.md` — these are existing languages
2. Scan `README.*.md` files to discover additional languages
3. Chinese (zh) always exists as the source language in the root directory

---

## Mode 1: Status Check (no arguments)

### Steps

For each discovered language, check the sync status of its translation files:

```bash
# For each mapping pair, execute:

# 1. Get the last commit that modified the translation file
TRANS_COMMIT=$(git log -1 --format=%H -- "<translation_file_path>")

# 2. Check if source file has been modified since then
BEHIND=$(git log --oneline "$TRANS_COMMIT"..HEAD -- "<source_file_path>" | wc -l)

# 3. Determine status
# BEHIND = 0 → ✅ In sync
# BEHIND > 0 → ⚠️ Outdated (behind by N commits)
# Translation file doesn't exist → ❌ Missing
```

### Output Format

```
📋 i18n Sync Status Report
═══════════════════════════

Source language: zh (root directory)
Supported languages: en

📁 en/ — English
  ✅ CLAUDE.md                           (in sync @ <short_hash>)
  ⚠️  ai-engineering-leveling-guide.md    (outdated: behind by 3 commits)
  ✅ PROGRESS.md                         (in sync @ <short_hash>)
  ✅ commands/coach/assess.md             (in sync)
  ⚠️  commands/coach/practice.md          (outdated: behind by 1 commit)
  ✅ commands/coach/install.md            (in sync)
  ✅ commands/coach/progress-report.md    (in sync)
  ✅ commands/coach/review-prompt.md      (in sync)

Summary: 6/8 in sync, 2 outdated
→ Run `/coach:i18n sync en` to sync outdated translations
```

## Mode 2: Sync Translations (`sync` / `sync <lang>`)

### Steps

#### Step 1: Detect Outdated Files

Use the "Status Check" git logic to collect all outdated and missing files.

#### Step 2: Get Change Content

For each outdated file:

```bash
# Get the commit when the translation was last updated
TRANS_COMMIT=$(git log -1 --format=%H -- "<translation_file_path>")

# Get the source file diff since then
git diff "$TRANS_COMMIT" -- "<source_file_path>"
```

#### Step 3: Decide Sync Strategy

- **1 file outdated** → Translate directly on the current branch
- **2+ files outdated** → Use git worktree for parallel translation:

```bash
# Create a worktree for each large file or file group
git worktree add ../i18n-sync-<name> -b i18n-sync/<name>

# After translation, merge back
git merge i18n-sync/<name>
git worktree remove ../i18n-sync-<name>
```

#### Step 4: Execute Translation

**Translation rules** (very important):

1. **Incremental translation**: Only translate portions affected by the diff — leave existing translations unchanged
2. **CLAUDE.md special handling**: Prompt upgrade examples need localized rewriting, not literal translation
   - Chinese examples like "用 useState 加个搜索框" → English uses equivalent English prompt scenarios
3. **Technical terms**: Keep in original form (useState, git worktree, CLAUDE.md, etc.)
4. **Code blocks**: Only translate comments, don't modify code
5. **Markdown formatting**: Maintain exactly the same structure as the source file
6. **README translation direction is reversed**: README.md (English) → README.<lang>.md

#### Step 5: Commit

One commit per translation file, format:

```
i18n(<lang>): sync <filename>

Translated changes from <source_commit_short>..<head_commit_short>

Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>
```

#### Step 6: Output Results

```
📋 i18n Sync Complete
═════════════════════

✅ en/ai-engineering-leveling-guide.md — synced (translated 3 changed paragraphs)
✅ en/commands/coach/practice.md — synced (translated 1 new practice item)

Summary: 2 files synced, 0 failed
```

## Mode 3: Add New Language (`add <lang>`)

### Steps

#### Step 1: Validate

- Confirm `<lang>` is a valid language code (e.g., ja, ko, es, fr, de)
- Confirm `<lang>/` directory doesn't already exist (avoid overwriting)

#### Step 2: Create Directory Structure

```bash
mkdir -p <lang>/commands
```

#### Step 3: Create README

Create `README.<lang>.md`, including:
- Language switcher links at the top (linking to all existing language READMEs)
- Content translated from README.md

#### Step 4: Update Language Switcher Links in Existing READMEs

Update `README.md`, `README.zh.md`, and other existing `README.<lang>.md` files to add the new language link in the language switcher bar.

#### Step 5: Translate All Files in Parallel

Use git worktree for parallel translation:

```
Worktree 1: <lang>/CLAUDE.md
Worktree 2: <lang>/ai-engineering-leveling-guide.md (largest file)
Worktree 3: <lang>/commands/*.md + <lang>/PROGRESS.md (small files combined)
```

Translation rules same as Mode 2, Step 4.

#### Step 6: Update Install Scripts

In `scripts/install.sh` and `scripts/install.ps1`:
- Add the new language to the `--lang` argument validation list

#### Step 7: Commit and Output

```
📋 New Language Added: <lang>
═══════════════════════════

Created:
  📁 <lang>/
  📄 <lang>/CLAUDE.md
  📄 <lang>/PROGRESS.md
  📄 <lang>/ai-engineering-leveling-guide.md
  📄 <lang>/commands/ (5 command files)
  📄 README.<lang>.md

Updated:
  📄 README.md — added <lang> language switcher link
  📄 README.zh.md — added <lang> language switcher link
  📄 scripts/install.sh — supports --lang <lang>
  📄 scripts/install.ps1 — supports -Lang <lang>

→ Run `./scripts/install.sh --lang <lang>` to install the new language
→ Run `/coach:i18n` to check sync status
```
