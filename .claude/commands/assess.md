对用户进行全面的 AI 工程能力评估。

## 评估流程

### Step 1：读取当前状态
先读取项目根目录的 `PROGRESS.md`，了解用户的历史进度和当前聚焦。

### Step 2：逐项评估（基于 ai-engineering-leveling-guide.md 的自我评估 Checklist）

对以下每项打分（0 = 完全不符合，1 = 部分符合，2 = 完全符合）：

**基础能力（Level 1-2）**
- 日常开发中使用 AI 代码补全
- 会在 Claude Code 中进行基本对话
- 能判断 AI 输出的代码是否可用

**提示词能力（Level 3-4）**
- 能写包含上下文+约束+期望输出的结构化 Prompt
- 项目有 CLAUDE.md 且保持更新
- 复杂任务会先用 Plan Mode 规划
- AI 生成的代码需手动修改的比例 < 20%

**自主开发（Level 5）**
- 描述业务意图而非技术实现，AI 自主选择方案
- 能独立委托完整 Feature（前端+API+测试）
- AI 的方案一次通过架构审查的比例 > 80%

**并行与编排（Level 6-7）**
- 能同时管理 3+ 个 Claude Code 实例并行开发
- 熟练使用 Git Worktree 隔离并行任务
- 有 5+ 个 Custom Slash Commands
- 配置了 Hooks 自动化质量检查

**系统级（Level 8）**
- CI/CD 中集成了 AI（Headless 模式）
- 有 AI 自动修复失败 Pipeline 的能力
- 有完整的 AI 工作流监控和成本控制

### Step 3：计算得分与 Level

评分标准：0-4 → Level 1-2 ｜ 5-8 → Level 3-4 ｜ 9-11 → Level 5 ｜ 12-15 → Level 6-7 ｜ 16-18 → Level 8

### Step 4：对比历史

将本次评估结果与 PROGRESS.md 中的上次评估对比，指出：
- 哪些项目有进步（分数提升）
- 哪些项目停滞（与上次相同）
- 哪些项目需要重点突破

### Step 5：输出结果

格式：
```
## 评估结果

**日期**：[今天日期]
**总分**：X / 18
**当前 Level**：N

### 各维度得分
| 维度 | 得分 | 备注 |
|------|------|------|
| 基础能力 | X/6 | ... |
| 提示词能力 | X/8 | ... |
| 自主开发 | X/6 | ... |
| 并行与编排 | X/8 | ... |
| 系统级 | X/6 | ... |

### Level 8 子技能状态
[逐项列出 5 个子技能的当前状态和建议]

### 下一步建议
[基于评估结果，给出 1-3 个具体的下一步行动]
```

### Step 6：提示更新 PROGRESS.md

评估完成后，提醒用户更新 PROGRESS.md 中的评估日期和相关状态。如果用户同意，帮助他更新文件。
