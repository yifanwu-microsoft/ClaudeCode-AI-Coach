项目国际化（i18n）管理工具。检测翻译同步状态、同步过时翻译、添加新语言支持。

参数：$ARGUMENTS

## 参数解析

根据 `$ARGUMENTS` 判断执行模式：

- **空**（无参数）→ 执行「状态检查」
- **`sync`** → 同步所有语言的过时翻译
- **`sync <lang>`**（如 `sync en`）→ 只同步指定语言
- **`add <lang>`**（如 `add ja`）→ 添加新语言支持

---

## 文件映射表

源文件（中文，根目录）与翻译文件的对应关系：

| 源文件 | 翻译文件 |
|--------|---------|
| `CLAUDE.md` | `<lang>/CLAUDE.md` |
| `PROGRESS.md` | `<lang>/PROGRESS.md` |
| `ai-engineering-leveling-guide.md` | `<lang>/ai-engineering-leveling-guide.md` |
| `.claude/commands/assess.md` | `<lang>/commands/assess.md` |
| `.claude/commands/install.md` | `<lang>/commands/install.md` |
| `.claude/commands/practice.md` | `<lang>/commands/practice.md` |
| `.claude/commands/progress-report.md` | `<lang>/commands/progress-report.md` |
| `.claude/commands/review-prompt.md` | `<lang>/commands/review-prompt.md` |
| `.claude/commands/i18n.md` | （不翻译，项目维护工具） |

**README 特殊处理**：
- `README.md`（英文）是 README 的源文件
- `README.zh.md`（中文）、`README.<lang>.md`（其他语言）是翻译

---

## 语言发现

执行以下步骤发现已支持的语言：

1. 扫描根目录下的子目录，找到包含 `CLAUDE.md` 的目录作为已有语言
2. 扫描 `README.*.md` 文件发现更多语言
3. 中文（zh）始终作为源语言存在于根目录

## 模式 1：状态检查（无参数）

### 执行步骤

对每个已发现的语言，检查其翻译文件的同步状态：

```bash
# 对每个映射对，执行：

# 1. 获取翻译文件最后修改的 commit
TRANS_COMMIT=$(git log -1 --format=%H -- "<翻译文件路径>")

# 2. 检查源文件在此之后是否有修改
BEHIND=$(git log --oneline "$TRANS_COMMIT"..HEAD -- "<源文件路径>" | wc -l)

# 3. 判断状态
# BEHIND = 0 → ✅ 已同步
# BEHIND > 0 → ⚠️ 过时（落后 N 个 commit）
# 翻译文件不存在 → ❌ 缺失
```

### 输出格式

```
📋 i18n 同步状态报告
═══════════════════════

源语言：zh（根目录）
已支持语言：en

📁 en/ — English
  ✅ CLAUDE.md                           (已同步 @ <短hash>)
  ⚠️  ai-engineering-leveling-guide.md    (过时：落后 3 个 commit)
  ✅ PROGRESS.md                         (已同步 @ <短hash>)
  ✅ commands/assess.md                  (已同步)
  ⚠️  commands/practice.md               (过时：落后 1 个 commit)
  ✅ commands/install.md                 (已同步)
  ✅ commands/progress-report.md         (已同步)
  ✅ commands/review-prompt.md           (已同步)

汇总：6/8 已同步，2 个过时
→ 执行 `/i18n sync en` 同步过时翻译
```

## 模式 2：同步翻译（`sync` / `sync <lang>`）

### 执行步骤

#### Step 1：检测过时文件

使用「状态检查」的 git 逻辑，收集所有过时和缺失的文件列表。

#### Step 2：获取变更内容

对每个过时的文件：

```bash
# 获取翻译文件最后更新时的 commit
TRANS_COMMIT=$(git log -1 --format=%H -- "<翻译文件路径>")

# 获取源文件自那之后的 diff
git diff "$TRANS_COMMIT" -- "<源文件路径>"
```

#### Step 3：决定同步策略

- **1 个文件过时** → 直接在当前分支翻译
- **2+ 个文件过时** → 使用 git worktree 并行翻译：

```bash
# 为每个大文件或文件组创建 worktree
git worktree add ../i18n-sync-<name> -b i18n-sync/<name>

# 翻译完成后合并回来
git merge i18n-sync/<name>
git worktree remove ../i18n-sync-<name>
```

#### Step 4：执行翻译

**翻译规则**（非常重要）：

1. **增量翻译**：只翻译 diff 涉及的部分，保留现有翻译不变
2. **CLAUDE.md 特殊处理**：prompt 升级示例需要本地化重写，不是直译
   - 中文示例如"用 useState 加个搜索框"→ 英文用等效的英文 prompt 场景
3. **技术术语**：保持原文（useState, git worktree, CLAUDE.md 等）
4. **代码块**：只翻译注释，不改代码
5. **Markdown 格式**：保持与源文件完全一致的结构
6. **README 翻译方向相反**：README.md（英文）→ README.<lang>.md

#### Step 5：提交

每个翻译文件一个 commit，格式：

```
i18n(<lang>): sync <文件名>

Translated changes from <source_commit_short>..<head_commit_short>

Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>
```

#### Step 6：输出结果

```
📋 i18n 同步完成
═══════════════

✅ en/ai-engineering-leveling-guide.md — 已同步（翻译了 3 个变更段落）
✅ en/commands/practice.md — 已同步（翻译了 1 个新增练习项）

汇总：2 个文件已同步，0 个失败
```

## 模式 3：添加新语言（`add <lang>`）

### 执行步骤

#### Step 1：验证

- 确认 `<lang>` 是有效的语言代码（如 ja, ko, es, fr, de）
- 确认 `<lang>/` 目录不存在（避免覆盖）

#### Step 2：创建目录结构

```bash
mkdir -p <lang>/commands
```

#### Step 3：创建 README

创建 `README.<lang>.md`，包含：
- 顶部语言切换链接（链接到所有已有语言的 README）
- 内容翻译自 README.md

#### Step 4：更新现有 README 的语言切换链接

更新 `README.md`、`README.zh.md` 和其他已有的 `README.<lang>.md`，在语言切换栏中添加新语言的链接。

#### Step 5：并行翻译所有文件

使用 git worktree 并行翻译：

```
Worktree 1: <lang>/CLAUDE.md
Worktree 2: <lang>/ai-engineering-leveling-guide.md（最大文件）
Worktree 3: <lang>/commands/*.md + <lang>/PROGRESS.md（小文件合并）
```

翻译规则同「模式 2」的 Step 4。

#### Step 6：更新安装脚本

在 `scripts/install.sh` 和 `scripts/install.ps1` 中：
- 添加新语言到 `--lang` 参数的验证列表

#### Step 7：提交并输出

```
📋 新语言添加完成：<lang>
═══════════════════════

已创建：
  📁 <lang>/
  📄 <lang>/CLAUDE.md
  📄 <lang>/PROGRESS.md
  📄 <lang>/ai-engineering-leveling-guide.md
  📄 <lang>/commands/ (5 个命令文件)
  📄 README.<lang>.md

已更新：
  📄 README.md — 添加了 <lang> 语言切换链接
  📄 README.zh.md — 添加了 <lang> 语言切换链接
  📄 scripts/install.sh — 支持 --lang <lang>
  📄 scripts/install.ps1 — 支持 -Lang <lang>

→ 执行 `./scripts/install.sh --lang <lang>` 安装新语言
→ 执行 `/i18n` 查看同步状态
```
